## What Has Been Tried

The early computational corpus established the shape of the problem but did not
settle it. The Python `mpmath` result `2919` was diagnosed as a finite
precision deduplication artifact. The Wolfram result `2926` remains the
strongest single-source computation, but it is not a proof certificate.

The local probe-refined table gives a strong candidate partition:

- the strict table has 2925 classes among 5139 candidates;
- a stable probe split separates candidate `25` from `{1404, 4239}`;
- the refined table has 2926 classes;
- candidate `57` remains the special overflow singleton.

In Lean, the most productive approach so far has been symbolic reduction of
small closed sublanguages generated by `i`, `i^i`, and `i^(i^i)`. This has
proved:

- all fourteen retained representatives in the near-`i^i` probe class are
  exactly `i^i = exp(-pi/2)`;
- the near-zero doubleton `{2207, 3777}` is an exact merge;
- the retained near-one pair `{1404, 4239}` is an exact merge;
- candidate `25` is separated from `{1404, 4239}` if a chain of explicit
  real endpoint estimates is discharged.

The candidate-25 split has been pushed down to scalar real inequalities. In
particular, Lean now proves the rational boxes for:

- `pi/2`, using mathlib's 20-decimal `pi` certificate;
- `rho = exp(-pi/2)`, using mathlib's 20-decimal `exp 1` certificate plus a
  Taylor bound for the residual `exp(0.57079632679...)`;
- `theta = (pi/2) * rho`, by interval multiplication from the two boxes above.

The current symbolic endpoint chain stops at the four trigonometric endpoint
inequalities needed to box `sin theta` and `cos theta`:

```lean
(320764449975 : Real) / 10^12 < sin (326536474946 / 10^12)
sin (326536474949 / 10^12) < (320764449985 : Real) / 10^12
(947158998071 : Real) / 10^12 < cos (326536474949 / 10^12)
cos (326536474946 / 10^12) < (947158998073 : Real) / 10^12
```

Immediately before pausing, a scratch Lean prototype showed a promising route
for these endpoint estimates using:

- `Real.hasSum_sin` and `Real.hasSum_cos`;
- `HasSum.tendsto_sum_nat`;
- `Antitone.alternating_series_le_tendsto` and
  `Antitone.tendsto_le_alternating_series`;
- simple antitonicity of the nonnegative Taylor magnitudes for `0 <= x <= 1`.

The prototype was intentionally not committed as proof progress. Numerically,
the necessary partial sums are small: 6 sine terms for the lower sine bound,
5 sine terms for the upper sine bound, 6 cosine terms for the lower cosine
bound, and 7 cosine terms for the upper cosine bound.

## What Remains Open

There is still no Lean theorem `a198683 12 = 2926`.

The main missing pieces are:

- discharge the trigonometric endpoint estimates for `sin theta` and
  `cos theta`;
- continue the same endpoint-certificate style through the later exponential,
  sine, and cosine boxes in the candidate-25 split;
- turn the proved split/merge facts into a semantic lower/upper bound that
  matches the 2926-class probe partition;
- finish the overflow candidate `57` argument semantically, probably via
  log-modulus/real-part bounds rather than direct argument reduction;
- connect any generated candidate-expression table back to the shared
  canonical `PowTower.Expr` definition rather than to an unproved evaluator.

The data certificate remains exactly that: it verifies the retained table, not
the mathematical equality relation on all 5139 principal-power values.

## Promising Future Directions

The highest-leverage next step is to add reusable Taylor endpoint lemmas for
`sin` and `cos` on `0 <= x <= 1`, using the alternating-series theorems already
available in mathlib. These lemmas should produce rational endpoint
inequalities by `norm_num` after the partial-sum bound is selected.

After that, the same pattern should be generalized to the later endpoint
obligations in `A198683N12Symbolic.lean`: keep every complex step symbolic,
reduce each remaining separation to monotone real interval arithmetic, and
discharge only the final rational endpoint inequalities by series bounds.

For candidate `57`, the most promising route is the one already suggested by
`A198683N12Magnitude` and `A198683N12OverflowWitness`: prove separation by
real-part/log-modulus inequalities. Avoid any strategy that requires reducing
an imaginary part of magnitude around `10^41232...` modulo `2*pi`.

For the global `a198683 12 = 2926` theorem, the eventual certificate should
probably combine three layers:

1. A generated but Lean-checkable map from all 5139 `n = 12` parenthesizations
   to shared `PowTower.Expr` syntax.
2. A library of exact symbolic reductions and interval separation witnesses
   for the exceptional clusters.
3. A finite graph/partition checker whose input certificates are proved to
   cover every same-class merge and every cross-class separation.

This would preserve the desired definition discipline: the shared lexical
definition remains primary, and every faster or more structured computation is
accepted only after a Lean proof connects it back to that definition.
